# 🛡️ CIAM Operations Automation (CIAM 운영 자동화)

통합회원(CIAM) 서비스 운영 중 발생하는 반복적이고 소모적인 수동 업무를 AI와 스크립트로 자동화하는 프로젝트입니다.

---

## 📂 1. 과제 목록 (Modules)

현재 진행 중인 과제와 향후 로드맵입니다. 각 폴더를 클릭하여 상세 가이드를 확인하세요.

* **[weekly-meeting](./weekly-meeting/): [진행 중]** 주간회의록 자동 생성 및 캘린더 연동
* **release-notes: [대기]** Jira 이슈 기반 릴리즈 노트 자동 생성
* **translation-sync: [대기]** 다국어 번역 메시지 관리 자동화

---

## 🚀 2. [과제 1] 주간회의 자동화 가이드

매주 금요일 오전 10시, 차주 월/화요일 회의 준비를 자동으로 마칩니다.

### 🏁 1단계: 준비하기 (Setup)
본격적인 코드 작성 전, 필요한 '열쇠'를 모으는 단계입니다.
1.  **Confluence 토큰 발급:** 프로필 설정에서 `CIAM_Weekly_Report`라는 이름으로 토큰을 발급받으세요.
2.  **부모 페이지 ID 확인:** 회의록이 생성될 상위 페이지의 주소창에서 `pageId=숫자`를 메모하세요.
3.  **구글 캘린더 ID 확인:** 대상 캘린더 설정에서 `c_...`로 시작하는 ID를 복사하세요.

### 🛠 2단계: 코드 설치 (Implementation)
1.  **Google Apps Script(GAS) 접속:** [script.google.com](https://script.google.com)에서 새 프로젝트를 만듭니다.
2.  **보안 속성 등록:** `설정(⚙️)` -> `스크립트 속성`에 `CIAM_Weekly_Report` 이름으로 토큰 값을 저장합니다.
3.  **코드 복사:** 본 저장소의 `weekly-meeting/main.gs` 내용을 GAS 에디터에 붙여넣습니다.
4.  **서비스 추가:** 왼쪽 `서비스 +`를 눌러 `Google Calendar API`를 추가합니다.

### ⚙️ 3단계: 자동 실행 설정 (Automation)
1.  GAS 왼쪽 메뉴의 **'트리거(⏰)'** 아이콘을 클릭합니다.
2.  `새 트리거 추가`를 누르고 아래와 같이 설정합니다.
    * **실행할 함수:** `main` (또는 메인 함수명)
    * **이벤트 소스:** `시간 기반`
    * **타이머 유형:** `주간 타이머`
    * **요일/시간:** `매주 금요일 / 오전 10시~11시`

---

## ⚠️ 주의사항 (Caution)

1.  **보안 제일:** 어떤 경우에도 API 토큰을 소스코드(GitHub)에 직접 적지 마세요. 반드시 **스크립트 속성**을 이용합니다.
2.  **내부망 체크:** 사내 컨플루언스 API 주소가 외부(Google 서버)에서 호출 가능한지 확인이 필요합니다. (타 팀 사례 참고)
3.  **날짜 변동:** 공휴일 등으로 회의가 취소될 경우, 캘린더에 이벤트가 없으면 스크립트는 동작하지 않도록 설계되었습니다.
